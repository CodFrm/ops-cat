# OpsKat 20s Feature Promo Expanded Prompt

## Title + Style

Create a 20-second 16:9 feature promo for OpsKat, an open-source AI-first desktop operations tool. Use the repository's existing logo and screenshots as proof. Palette: `#071016`, `#101923`, `#0B121A`, `#EEF4F7`, `#AAB5BD`, `#4D7CFE`, `#58D980`, `#F5C04A`, `#F07067`, `#243241`. Typography: `Bahnschrift` for display/data, `Microsoft YaHei UI` for Chinese copy, `Cascadia Code` for commands.

## Rhythm

hook-build-PEAK-proof-CTA. Five scenes across 20 seconds: 0-4.2, 3.7-7.9, 7.4-11.6, 11.1-15.6, 15.2-20.0. Each scene overlaps the next by about 0.4-0.5 seconds through a blur push and block wipe transition.

## Global Rules

Use existing product screenshots in moving device frames rather than abstract illustrations. Keep the UI dark and operational, but use green, amber, and coral for security decisions, approvals, and blocked actions. Every scene needs background texture, midground product proof, foreground labels or metrics, and entrance animations for all visible elements.

## Scene 1: Hook

Concept: OpsKat appears as the control surface where natural language turns into infrastructure action. The logo and product screenshot establish this as a real desktop app, not a generic AI claim.

Depth layers: BG radial glows, ghost ops terms, faint grid. MG large Chinese headline and screenshot device frame. FG logo badge, prompt pill, platform labels.

Choreography: logo lifts in, headline stamps in from the left, prompt pill types on visually, screenshot slides in with perspective, platform chips cascade upward.

Transition out: brand-blue/green block wipe plus incoming blur push.

## Scene 2: Unified Assets

Concept: The viewer sees SSH, Redis, MongoDB, MySQL/PGSQL, Kafka, and K8s clusters unified in one workspace. The screenshot becomes a living map of infrastructure.

Depth layers: BG asset graph, screenshot window, feature chips, side labels, metric strips.

Choreography: screenshot pushes forward, chips drop in around the frame, connection lines draw across the UI, metric counters rise.

Transition out: velocity push to the AI execution scene.

## Scene 3: AI Executes

Concept: A natural-language request triggers real operational steps: SSH command, database query, Redis/Kafka inspection. The AI is useful because it can chain actions under the same desktop context.

Depth layers: BG terminal scan lines, MG request bubble and execution cards, FG progress path and result summary.

Choreography: request slides in, command cards cascade, path line draws, result badge counts up, assistant response expands.

Transition out: staggered block wipe into safety proof.

## Scene 4: Policy + Audit

Concept: Safety is not an afterthought; approvals, policies, parser checks, and audit logs are first-class. The frame should feel like a control room gate.

Depth layers: BG lock rings and audit grid, MG policy gate panel, FG allow/deny chips, approval queue, audit trail.

Choreography: shield scales in, policy rows sweep in, green approvals tick on, blocked danger command flashes coral then settles, audit entries fill downward.

Transition out: calm blur crossfade to CTA.

## Scene 5: CTA

Concept: Resolve to the product identity and clear use case: open-source desktop operations with opsctl for AI coding tools. Keep the final frame readable and restrained.

Depth layers: BG orbit rings and screenshot silhouettes, MG logo and claim, FG feature chips, website/github line.

Choreography: logo floats in, title anchors, chips cascade, opsctl command appears in a terminal strip, final black overlay fades in at the end.

## Negative Prompt

Avoid generic SaaS landing-page hero layouts, abstract purple gradients, hand-wavy AI visuals, tiny web UI text, and any scene that lacks product evidence.
